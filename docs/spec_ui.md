# UI / UX 仕様（編集画面 & ライブ画面）

## 1. 画面レイヤー

1. 編集画面（Edit View）
2. ライブパフォーマンス画面（Live Performance View）
3. ソングモード画面（Song View）

### 1.1 編集画面

- 16step × 2小節グリッドを中心にしたステップシーケンサー。
- ステップタップ：ノート ON/OFF
- ロングタップ：オーナメント編集画面に遷移（顔変形 UI）。
- 上部: パターン選択（16 × 8 バンク）、トラック選択、timeScale、GrvDepth。
- 下部: Live Take 編集タブ（ピアノロールに近いビューで 2 小節単位の編集）。

### 1.2 Live Performance View

- 上段：トラック Mute / Solo、スワップスタイルボタン（A〜E）、Step Loop ボタン群。
- 中段：
  - 中央に Swap Fader（0〜100%）。
  - 両側にレーン別 Ornament ノブ、GrvDepth ノブ、Step Length / Shift ノブ。
- 下段：MINI シーケンサー（簡易グリッド）。
  - タップでノートミュート。
  - 装飾有りステップは ◎ 表示で点滅。

### 1.3 ソングモード画面

- タイムライン形式で「パターン番号」と「パターンの繰り返し」を可視化。
- ドラム / ベース / リード / Live Take トラックが横に並び、  それぞれのパターンシーケンスを編集可能。
- 「ライブ中のパターン切り替えを録音」して Song View に反映するモードを持つ。
