# Platform Notes (Android / Windows Tablet) — Draft

ここでは、将来的に実装する際に考慮すべき
Android タッチデバイスと Windows タブレットの差異をメモしておく。

## 1. 入力デバイス

### Android

- マルチタッチ前提（2〜4本指）
- 画面サイズはスマホ〜タブレットまで幅広い
- 物理キーボードは基本期待しない
- 長押し／スワイプ／ピンチがUIの要

### Windows タブレット

- タッチ + ペン + キーボード + マウスの併用を想定
- マウス操作前提の小さな UI パーツも許容される
- キーボードショートカットで：
  - パターン切替
  - ミュート／ソロ
  - Live Take REC/PLAY
  を高速に操作できると便利

## 2. レイアウト上の違い

- Android：
  - ライブ画面と編集画面は「縦方向ページ」切り替えがメイン
  - ミニシーケンサーは高さを抑えつつタッチしやすいサイズで
- Windows：
  - 横長比率が多いので、左にシーケンサー、右にライブコントロールなど
    2ペインレイアウトも検討可能

## 3. パフォーマンス

- Android：
  - 音源エンジンの負荷にシビア
  - オーナメント最大イベント数やポリフォニーに制限を設ける
- Windows：
  - 多少重い処理でも動く余地はあるが、
    コードは出来るだけ共通ロジックにしておく

## 4. ファイル保存と共有

- 両プラットフォームで：
  - `project.json`
  - `patterns.json`
  - `livetakes.json`
  などの共通 JSON 形式で保存し、GitHub にそのまま置けるようにする。

- Android では内部ストレージ + SAF（Storage Access Framework）対応、
  Windows では通常のファイルダイアログ対応を検討。
